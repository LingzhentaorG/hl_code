# 架构说明

## 1. 总体结构

```text
CLI / batch_process.py / release_product.py
  -> ConfigManager / CRSManager
  -> InputManager
  -> PreprocessEngine
  -> GroundFilterEngine
  -> DEMEngine
  -> TileManager
  -> OutputManager
  -> compare_dem_results.py
```

仓库分层：

- `include/dem/`：公开头文件与核心类型
- `src/`：C++ 主产线实现
- `scripts/`：批处理、发布、GeoTIFF 写出、对比分析
- `tests/`：C++ doctest 与 Python 回归测试
- `config/`：正式配置、官方清单、参考 DEM 映射

## 2. 主处理流程

`MainController` 负责整条正式流水线的编排：

1. 读取配置并做路径归一化
2. 解析 CRS
3. 扫描输入点云并决定是否进入 Tile 模式
4. 预处理与地面滤波
5. 构建 `raw_direct`、`ground_direct`
6. 生成 `dem_object_mask`
7. 精炼 accepted support，得到 `dem_raw`
8. 由 `filtered_points` 生成 `dem_domain_mask`
9. 运行 ArcGIS 风格 KNN IDW，得到最终 `dem`
10. 生成 `dem_boundary_mask`
11. 输出栅格、日志和报告

## 3. 模块职责

`ConfigManager`

- 解析 JSON 配置
- 应用 `--set` 覆盖参数
- 归一化派生值
- 拒绝历史废弃字段

`InputManager`

- 校验 ASCII PLY
- 扫描点云统计摘要
- 读取全量点云或流式遍历点

`PreprocessEngine`

- 清洗 NaN/Inf
- 去重
- 去除极端异常点

`GroundFilterEngine`

- SOR 离群点过滤
- 地面种子提取与迭代扩张

`DEMEngine`

- 创建栅格模板
- 直落格最小高程栅格化
- 对象掩膜
- accepted support 精炼
- 域掩膜与边界掩膜
- KNN IDW 插值

`TileManager`

- 按内存阈值和网格尺寸切块
- 物化临时 `.xyz`
- 合并 tile 结果

`OutputManager`

- 写 GeoTIFF / PNG
- 写 `stats.txt` 与 `report.json`
- 统一正式目录结构

`compare_dem_results.py`

- 对齐主 DEM 与参考 DEM
- 输出范围一致性、高程误差、坡度差异、热点明细
- 产出 `comparison/` 和 `comparison_summary.*`

## 4. 配置归一化规则

当前配置统一使用 snake_case，并只保留正式产线字段。

关键规则：

- `ground.seed_grid_size <= 0`：自动置为 `3 * dem.cell_size`
- `ground.search_radius <= 0`：自动置为 `max(6 * cell_size, 2 * seed_grid_size)`
- `tile.tile_buffer <= 0`：自动置为 `max(search_radius, positive(idw_max_distance))`
- `dem.idw_max_distance <= 0`：在运行时按输出范围对角线解释

已删除字段在配置加载阶段直接报错，包括：

- `nearest_max_distance`
- `idw_mode`
- `idw_radius`
- `idw_max_points`
- `edge_shrink_cells`
- `fill_holes`
- `fill_max_radius`
- `fill_min_neighbors`

## 5. 路径与命名规范

- 对外配置项、脚本参数、JSON 报告字段统一采用 snake_case
- 数据集目录名使用原始数据集名，不再额外混入英文别名
- 正式发布目录固定为 `output/final_release/`
- 批处理比较输出固定为每数据集 `comparison/`
- C++ 侧统一通过 UTF-8 路径 helper 输出路径，避免 Windows 中文路径日志乱码

## 6. 测试体系

`build/dem_tests.exe`

- 配置解析与废弃字段拒绝
- 正式输出结构
- 凸包域、边界掩膜与 IDW 行为
- Tile 与批处理基础合同

`tests/test_compare_dem_results.py`

- 整像元窗口对齐回归
- 重投影对齐回归
- `batch_process.py` 的 compare-map / manifest / batch-name 集成行为
- 资源三号真实数据回归校验

## 7. 发布流程

正式发布脚本为 [release_product.py](../scripts/release_product.py)。

标准流程：

1. 清理 `build/`、`output/` 和缓存
2. 重新执行 `cmake -S . -B build`
3. 重新执行 `cmake --build build -j 4`
4. 运行 `dem_tests.exe`
5. 运行 `python tests/test_compare_dem_results.py`
6. 调用 `batch_process.py` 按官方 manifest 生成 `output/final_release/`
7. 再次运行 Python 对比测试并验证最终目录合同
