# 使用手册

## 1. 工具定位

当前版本只保留一条正式产线：

1. 输入验证与 CRS 解析
2. 预处理与离群点清洗
3. 地面点提取
4. 对象掩膜与支撑点精炼
5. `filtered_points` 凸包域生成
6. ArcGIS 风格 KNN IDW 重建官方 `DEM`
7. 输出正式栅格、掩膜、日志与对比分析

旧的最近邻 DEM、历史 IDW 变体、补洞产物、兼容别名文件名都已经下线。

## 2. 环境准备

安装依赖：

```powershell
pip install numpy rasterio matplotlib Pillow
```

构建：

```powershell
cmake -S . -B build
cmake --build build -j 4
```

## 3. 单次运行

```powershell
.\build\dem_cli.exe --config .\config\example_config.json
```

运行时覆盖参数：

```powershell
.\build\dem_cli.exe --config .\config\example_config.json --set dem.cell_size=4.0 --set ground.max_height_diff=0.6
```

默认单次输出：

```text
output/<timestamp>/
├── surface/
├── mask/
├── log/
└── pointcloud/          # 仅 output.write_debug_pointclouds=true 时生成
```

## 4. 官方批处理

推荐只通过官方清单执行正式批处理，避免误扫 `data/` 中的历史实验 PLY：

```powershell
python .\scripts\batch_process.py `
  --cli .\build\dem_cli.exe `
  --config .\config\example_config.json `
  --input-dir .\data `
  --output-root .\output `
  --manifest .\config\official_dataset_manifest.json `
  --batch-name final_release `
  --compare-map .\config\reference_dem_map.json `
  --jobs 1
```

参数说明：

- `--manifest`：只处理清单中的数据集，当前官方清单固定为 `基准`、`资源三号_石林_点云示范`、`高分七号_条带点云示范`
- `--batch-name`：使用稳定输出目录名，正式发布固定为 `final_release`
- `--compare-map`：按数据集名匹配参考 DEM；缺失映射时会在比较汇总中显式标记 `skipped`

## 5. DEM 对比分析

单独复跑对比：

```powershell
python .\scripts\compare_dem_results.py `
  --main-dem "D:\Desktop\hl_code\output\final_release\datasets\资源三号_石林_点云示范\surface\dem.tif" `
  --reference-dem "D:\Desktop\hl_code\arcgis\dem_idw_final.tif" `
  --mask-dir "D:\Desktop\hl_code\output\final_release\datasets\资源三号_石林_点云示范\mask" `
  --output-dir "D:\Desktop\hl_code\output\final_release\datasets\资源三号_石林_点云示范\comparison" `
  --dataset-name "资源三号_石林_点云示范"
```

比较模块行为：

- 主结果优先对齐
- 支持整像元窗口对齐和重投影对齐
- 范围一致性与高程误差分开统计
- 输出坡度一致性与异常热点
- 固定生成 `report.json`、`summary.md`、4 张图和 `hotspots.csv`

## 6. 正式发布

一键清理、重建、测试和生成正式结果：

```powershell
python .\scripts\release_product.py
```

该脚本会执行：

1. 清理 `build/`、`output/`、`config/output_tile_smoke/`、`__pycache__` 等生成物
2. 重新配置和构建
3. 运行 C++ 与 Python 测试
4. 按官方清单生成 `output/final_release/`
5. 再次运行 Python 对比测试并做最终结构验收

## 7. 输出解释

`surface/`：

- `dem.tif`：官方最终 DEM
- `dem_raw.tif`：accepted support 栅格

`mask/`：

- `dem_domain_mask`：`filtered_points` 二维凸包填充域
- `dem_boundary_mask`：凸包边界轮廓
- `dem_object_mask`：高起伏对象检测结果，仅用于支撑点剔除和 QC

`log/`：

- `run_log.txt`：执行日志
- `config_used.json`：解析并归一化后的实际配置
- `stats.txt`：计数、比例、耗时统计
- `report.json`：机器可读结果概览

`comparison/`：

- `report.json`：详细比较结果
- `summary.md`：人工可读评价
- `domain_overlap.png`、`diff_signed.png`、`diff_abs.png`、`slope_diff.png`
- `hotspots.csv`：异常点明细

## 8. 常见问题

为什么对象区没有被挖成 NoData 洞？

对象掩膜只负责剔除支撑点，不再从最终域中扣洞；最终高程由 IDW 在完整域内重建。

为什么 `idw_max_distance=0` 还能插值？

当前定义不是“零距离”，而是“自动模式”，会按输出范围对角线设置最大搜索距离。

为什么批处理结果里有 `comparison_summary` 但某些数据集是 `skipped`？

这是正常行为。只有在 `reference_dem_map.json` 中显式配置参考 DEM 的数据集才会执行对比。
