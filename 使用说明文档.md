## 1. 项目概述

### 1.1 功能定位

本工具是一个**命令行 DEM（数字高程模型）生成程序**，专门面向**光学测绘卫星摄影测量点云**数据处理。核心功能是从输入的 PLY 格式点云中提取地面点，并通过插值算法生成规则格网的高程栅格数据。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **输入格式** | ASCII PLY 点云文件（支持 XYZ + 可选强度/颜色） |
| **输出格式** | GeoTIFF（地理参考）、PNG（可视化）、PLY（中间点云） |
| **地面滤波** | 迭代式种子扩张算法（支持多种参考模式） |
| **离群点剔除** | 基于 KNN 统计的 3D 邻域离群点检测 |
| **插值方法** | 最近邻 / IDW（反距离加权）双模式可选 |
| **空洞填充** | 迭代式邻域均值填充 NoData 区域 |
| **分块处理** | Tile 模式自动切分大数据集（内存感知） |
| **配置管理** | JSON 配置 + 运行时参数覆盖 |
| **批量处理** | Python 脚本支持目录级批量作业 |

### 1.3 技术栈

- **编程语言**: C++20（强制要求）
- **构建系统**: CMake 3.24+
- **编译器**: MinGW g++（Windows）/ GCC / Clang（Linux）
- **空间索引**: nanoflann（KD-Tree）
- **JSON 解析**: nlohmann/json
- **GeoTIFF 后端**: rasterio（Python 脚本调用）
- **测试框架**: doctest

## 4. 构建指南

### 4.1 标准构建流程

```powershell
# 创建构建目录并配置
cmake -S . -B build -G "MinGW Makefiles"

# 编译（使用 4 个并行任务加速）
cmake --build build -j 4
```

构建完成后，可执行文件位于：
- `build/dem_cli.exe`（Windows）
- `build/dem_cli`（Linux）

### 4.2 高级构建选项

#### 启用/禁用 OpenMP 并行加速

```powershell
# 默认启用 OpenMP
cmake -S . -B build -DDEM_ENABLE_OPENMP=ON

# 禁用 OpenMP（调试或兼容性需求）
cmake -S . -B build -DDEM_ENABLE_OPENMP=OFF
```

#### 启用/禁用单元测试

```powershell
# 默认构建测试
cmake -S . -B build -DDEM_ENABLE_TESTS=ON

# 跳过测试构建以加快编译速度
cmake -S . -B build -DDEM_ENABLE_TESTS=OFF
```

#### 选择 GeoTIFF 后端

```powershell
# AUTO: 自动检测（默认优先 Python 后端）
cmake -S . -B build -DDEM_GTIFF_BACKEND=AUTO

# PYTHON: 强制使用 Python rasterio 后端
cmake -S . -B build -DDEM_GTIFF_BACKEND=PYTHON

# NATIVE: 使用原生 libtiff + PROJ（需预先安装相关开发包）
cmake -S . -B build -DDEM_GTIFF_BACKEND=NATIVE
```

### 4.3 IDE 集成

#### Visual Studio Code

- **C/C++ Extension Pack**：代码智能提示

## 5. 配置文件详解

配置文件采用 JSON 格式，示例见 `config/example_config.json`。以下是所有配置项的完整说明。

### 5.1 `input` - 输入文件配置

```json
{
  "input": {
    "file_path": "../data/基准.ply"
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `file_path` | string | （必填） | 输入的 ASCII PLY 文件路径（支持相对路径和绝对路径） |

**支持的 PLY 格式要求**：
- 仅支持 **ASCII** 格式（二进制 PLY 尚未实现）
- 必须包含 `x`, `y`, `z` 三个顶点属性
- 可选属性：`intensity`（强度）、`red/green/blue`（颜色）、`normal_x/y/z`（法向量）

### 5.2 `crs` - 坐标参考系配置

```json
{
  "crs": {
    "epsg_code": 4326,
    "allow_unknown_crs": false
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `epsg_code` | int | 4326 | EPSG 编码（4326=WGS84 经纬度） |
| `allow_unknown_crs` | bool | false | 是否允许未知 CRS 继续处理 |

**常用 EPSG 代码**：
- `4326`: WGS84 经纬度（度）
- `3857`: Web Mercator（米）
- `4490`: CGCS2000 经纬度（中国测绘常用）
- `4547`: CGCS2000 3-degree Gauss-Kruger zone 37（米）

⚠️ **重要提示**：生产环境建议将 `allow_unknown_crs` 设为 `false` 以确保地理参考正确性。

### 5.3 `preprocess` - 预处理参数

```json
{
  "preprocess": {
    "sample_spacing_count": 512,
    "max_extreme_sample": 5000
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `sample_spacing_count` | int | 512 | 用于估算平均点间距的采样点数 |
| `max_extreme_sample` | int | 5000 | 极端离群点剔除时的最大采样数 |

**预处理阶段执行的操作**：
1. **NaN/Inf 清洗**：移除包含非数值坐标的点
2. **去重**：移除完全重复的点（XYZ 完全相同）
3. **极端离群点剔除**：基于各坐标轴的分位数范围过滤明显异常点

### 5.4 `outlier` - KNN 离群点过滤参数

```json
{
  "outlier": {
    "knn": 16,
    "stddev_multiplier": 1.5,
    "use_robust_stat": false
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `knn` | int | 16 | 每个点的 K 近邻数量 |
| `stddev_multiplier` | float | 1.5 | 标准差倍数阈值 |
| `use_robust_stat` | bool | false | 是否使用鲁棒统计（中位数+MAD） |

**算法原理**：
- 对每个点查找其 K 个最近邻
- 计算该点到邻居的平均距离
- 若距离 > 全局均值 + multiplier × 标准差，则判定为离群点

**参数调优建议**：
- 点云密度高 → 增大 `knn`（如 20~30）
- 噪声较多 → 减小 `stddev_multiplier`（如 1.0~1.2）
- 存在大量异常值 → 启用 `use_robust_stat`

### 5.5 `ground` - 地面滤波核心参数 ⭐

这是**最关键**的配置段，直接影响地面点提取质量。

```json
{
  "ground": {
    "seed_grid_size": 0.0,
    "knn": 8,
    "search_radius": 0.0,
    "max_height_diff": 1.0,
    "max_slope_deg": 20.0,
    "max_iterations": 6,
    "min_ground_neighbors": 3,
    "reference_mode": "local_plane",
    "reference_fallback_mode": "inverse_distance_squared",
    "distance_weight_power": 2.0,
    "min_neighbor_completeness": 0.5
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `seed_grid_size` | float | 0.0 | 粗格网尺寸（0=自动计算） |
| `knn` | int | 8 | 地面点扩张的近邻数量 |
| `search_radius` | float | 0.0 | 搜索半径（0=自动计算） |
| `max_height_diff` | float | 1.0 | 最大允许高程差（米） |
| `max_slope_deg` | float | 20.0 | 最大允许坡角（度） |
| `max_iterations` | int | 6 | 最大迭代轮次 |
| `min_ground_neighbors` | int | 3 | 判定地面点的最小近邻数 |
| `reference_mode` | string | local_plane | 首选参考模式 |
| `reference_fallback_mode` | string | inverse_distance_squared | 回退参考模式 |
| `distance_weight_power` | float | 2.0 | IDW 权重距离幂次 |
| `min_neighbor_completeness` | float | 0.5 | 邻域完整度阈值 [0,1] |

**参考模式说明**：
- `local_plane`: 最二乘平面拟合（精度最高，计算量大）
- `inverse_distance_squared`: IDW 加权平均（平衡方案）
- `local_min`: 取最低点（最保守，适合平坦地形）

**参数调优建议**：

| 场景 | 推荐调整 |
|------|----------|
| 平坦城市区域 | `max_height_diff=0.3~0.5`, `max_slope_deg=10~15` |
| 山地丘陵 | `max_height_diff=1.5~2.5`, `max_slope_deg=25~35` |
| 密集植被区 | `max_iterations=8~10`, `knn=12~15` |
| 低密度点云 | `seed_grid_size` 手动设为较大值（如 10~20 米） |
| 复杂建筑群 | `min_ground_neighbors=5`, `min_neighbor_completeness=0.7` |

### 5.6 `dem` - DEM 插值参数

```json
{
  "dem": {
    "cell_size": 8.0,
    "interpolation_method": "nearest",
    "nearest_max_distance": 0.0,
    "idw_radius": 0.0,
    "idw_power": 2.0,
    "hole_fill_enabled": true,
    "hole_fill_max_iterations": 50,
    "hole_fill_threshold": 0.1,
    "nodata_value": -9999.0
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `cell_size` | float | 8.0 | DEM 栅格分辨率（米/像素） |
| `interpolation_method` | string | nearest | 插值方法（nearest/idw） |
| `nearest_max_distance` | float | 0.0 | 最近邻搜索半径（0=自动） |
| `idw_radius` | float | 0.0 | IDW 搜索半径（0=自动） |
| `idw_power` | float | 2.0 | IDW 距离幂次 |
| `hole_fill_enabled` | bool | true | 是否启用空洞填充 |
| `hole_fill_max_iterations` | int | 50 | 空洞填充最大轮次 |
| `hole_fill_threshold` | float | 0.1 | 收敛阈值（米） |
| `nodata_value` | float | -9999.0 | 无数据标记值 |

**插值方法选择**：

| 方法 | 适用场景 | 特点 |
|------|----------|------|
| `nearest` | 地面点密集均匀 | 速度快，保持原始值不变 |
| `idw` | 地面点分布不均 | 平滑过渡，但可能过度平滑陡坎 |

**分辨率选择建议**：
- 高精度制图：`cell_size=1.0~2.0`
- 中等精度：`cell_size=5.0~10.0`
- 概览/背景：`cell_size=20.0~50.0`

### 5.7 `tile` - 分块处理参数

```json
{
  "tile": {
    "mode": "auto",
    "target_points_per_tile": 2000000,
    "overlap_meters": 20.0,
    "aggregate_method": "merge",
    "force_tile_width_cells": 1000,
    "force_tile_height_cells": 1000
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `mode` | string | auto | 分块模式（auto/off/forced） |
| `target_points_per_tile` | int | 2000000 | 目标每块点数 |
| `overlap_meters` | float | 20.0 | 块间重叠宽度（米） |
| `aggregate_method` | string | merge | 结果聚合方式 |
| `force_tile_width_cells` | int | 1000 | 强制模式下每块列数 |
| `force_tile_height_cells` | int | 1000 | 强制模式下每块行数 |

**分块模式说明**：
- `auto`: 自动判断是否需要分块（点数超过目标值时触发）
- `off`: 禁用分块（适合小数据集）
- `forced`: 强制按指定网格尺寸分块（适合超大内存约束场景）

**何时需要分块**：
- 点云总点数 > 200 万
- 可用内存 < 8GB
- 处理超大面积（如整景卫星影像）

```json
{
  "output": {

```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `write_raw_ply` | bool | false | 输出原始点云 |
| `write_seed_ply` | bool | true | 输出种子点集 |
| `write_ground_ply` | bool | true | 输出地面点集 |
| `write_nonground_ply` | bool | true | 输出非地面点集 |

**PNG 渲染参数**（嵌套在 `write_png` 对象内）：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|

| `percentile_low` | int | 2 | 低端截断百分位 |

```powershell

```

### 6.3 运行时参数覆盖

通过 `--set` 参数可在不修改配置文件的情况下临时调整任何参数：

```powershell
# 示例1：调整 DEM 分辨率
--set dem.cell_size=2.0

# 示例2：切换插值方法
--set dem.interpolation_method=idw

# 示例3：调整地面滤波灵敏度
--set ground.max_height_diff=0.5
--set ground.max_slope_deg=15

# 示例4：禁用某项输出
--set output.write_nonground_ply=false

# 示例5：强制分块模式
--set tile.mode=forced
--set tile.force_tile_width_cells=500
```

**参数路径规则**：
- 使用 `.` 分隔嵌套层级
- 数组元素暂不支持直接索引
- 布尔值使用 `true/false`
- 字符串值无需引号（除非含空格）

## 7. 处理流程详解

### 7.1 完整流水线架构

```
┌─────────────┐
│  PLY 输入   │ ──→ InputManager（格式验证、统计）
└─────────────┘
       ↓
┌─────────────┐
│  CRS 解析   │ ──→ CRSManager（EPSG/WKT 解析）
└─────────────┘
       ↓
┌─────────────┐
│   预处理    │ ──→ PreprocessEngine（NaN清洗、去重、极端值剔除）
└─────────────┘
       ↓
┌─────────────┐
│ 离群点过滤  │ ──→ GroundFilterEngine::removeOutliers（KNN 统计）
└─────────────┘
       ↓
┌─────────────┐
│  地面滤波   │ ──→ GroundFilterEngine::extractGround（种子+迭代扩张）
└─────────────┘
       ↓
    ┌──┴──┐
    ↓     ↓
┌──────┐ ┌───────┐
│地面点│ │非地面点│
└──┬───┘ └───────┘
    ↓
┌─────────────┐
│  DEM 插值   │ ──→ DEMEngine::interpolate（最近邻/IDW）
└─────────────┘
       ↓
┌─────────────┐
│  空洞填充   │ ──→ DEMEngine::fillHoles（迭代均值填充）
└─────────────┘
       ↓
┌─────────────┐
│  结果输出   │ ──→ OutputManager（PLY/PNG/GeoTIFF/日志）
└─────────────┘
```

### 7.2 各阶段详情

#### 阶段 1：输入验证（InputManager）

- **功能**：读取 PLY 文件头，验证格式合规性
- **检查项**：
  - 文件是否存在且可读
  - 是否为 ASCII PLY 格式
  - 是否包含 x, y, z 属性
  - 点数是否合理（>0）
- **输出**：点云统计信息（边界框、点数、属性列表）

#### 阶段 2：CRS 解析（CRSManager）

- **功能**：确定坐标参考系定义
- **支持格式**：EPSG 编码、WKT 字符串、PROJ.4 字符串
- **输出**：标准化 CRS 定义（传递给 GeoTIFF 写入模块）

#### 阶段 3：预处理（PreprocessEngine）

- **步骤 3.1 - NaN/Inf 清洗**
  - 扫描所有点的坐标值
  - 移除包含 NaN 或 Inf 的点
  
- **步骤 3.2 - 去重**
  - 基于 XYZ 三维坐标精确匹配
  - 仅保留首次出现的点
  
- **步骤 3.3 - 极端离群点剔除**
  - 在每个坐标轴上计算分位数范围（Q1 - 3×IQR, Q3 + 3×IQR）
  - 超出范围的点被视为极端异常值移除

#### 阶段 4：离群点过滤（GroundFilterEngine::removeOutliers）

- **算法**：基于 KNN 的统计离群点去除（Statistical Outlier Removal, SOR）
- **流程**：
  1. 为每个点建立 KD-Tree 空间索引
  2. 查找每个点的 K 个最近邻
  3. 计算该点到邻居的平均距离 d_i
  4. 计算全局距离均值 μ 和标准差 σ
  5. 若 d_i > μ + multiplier × σ，则标记为离群点
- **鲁棒模式**：使用中位数和中位数绝对偏差（MAD）替代均值和标准差

#### 阶段 5：地面滤波（GroundFilterEngine::extractGround）

**子阶段 5.1 - 种子点选取**

- 将点云投影到 XY 平面
- 按 `seed_grid_size` 划分粗格网
- 在每个格网单元内选取 Z 值最低的点作为候选种子
- 过滤掉明显偏高（可能是建筑物/树木顶部）的候选点
- 最终得到初始地面种子点集

**子阶段 5.2 - 迭代式地面点扩张**

```
for iteration = 1 to max_iterations:
    for each unclassified point p:
        1. 在已确认地面点中查找 K 近邻
        2. 根据 reference_mode 计算 p 的参考高程 h_ref
        3. 计算高度差 Δh = p.z - h_ref
        4. 若 Δh ≤ max_height_diff 且坡角 ≤ max_slope_deg:
             标记 p 为地面点
        else:
             标记 p 为非地面点（若满足邻域完整性要求）
    
    若本轮无新地面点加入 → 提前终止
```

**参考高程计算方式**：

| 模式 | 计算公式 | 适用场景 |
|------|----------|----------|
| `local_plane` | 最小二乘平面拟合 | 精度优先 |
| `inverse_distance_squared` | Σ(w_i × z_i) / Σ(w_i), w_i = 1/d²² | 通用 |
| `local_min` | min(z_1, z_2, ..., z_k) | 保守策略 |

#### 阶段 6：DEM 插值（DEMEngine::interpolate）

- **输入**：地面点云 + 配置参数
- **输出**：二维高程栅格矩阵（DEMRaster）

**最近邻插值**：
- 对每个栅格中心点，查找最近的地面点
- 直接复制该点的高程值
- 若最近距离 > `nearest_max_distance`，标记为 NoData

**IDW 插值**：
- 对每个栅格中心点，查找半径内的所有地面点
- 按距离加权平均：h = Σ(w_i × z_i) / Σ(w_i)，其中 w_i = 1/d^p
- 若半径内无点，标记为 NoData

#### 阶段 7：空洞填充（DEMEngine::fillHoles）

- **目的**：填补因数据缺失导致的 NoData 区域
- **算法**：迭代式邻域均值填充
  ```
  while NoData 单元格存在且 iterations < max_iterations:
      for each NoData cell (i,j):
          查找 8-邻域中的有效值单元格
          若有效邻居数 ≥ 阈值:
              cell(i,j) = mean(neighbor_values)
      若本轮更新量 < threshold → 收敛退出
  ```

#### 阶段 8：结果输出（OutputManager）

根据配置决定输出哪些文件类型：

| `filtered.ply` | `write_filtered_ply=true` | 离群点剔除后的点云 |
| `seed.ply` | `write_seed_ply=true` | 地面滤波种子点集 |

## 9. 算法原理

### 9.1 地面滤波算法详解

本工具采用的地面滤波算法属于**迭代式形态学方法**的变体，结合了 TIN（不规则三角网）和 KD-Tree 空间查询的优势。

#### 9.1.1 为什么不用简单的全局阈值？

传统的"低于某高程即为地面"的方法存在致命缺陷：
- ❌ 无法处理起伏地形（山坡上的地面点高于谷底的建筑）
- ❌ 对噪声敏感（单个低矮异常点会拉低局部阈值）
- ❌ 不适应不同密度区域（稀疏区域的统计特性不同）

本算法的优势：
- ✅ **局部自适应**：每个点的判定基于其邻域特征
- ✅ **迭代精化**：多轮扩张逐步扩大已知地面范围
- ✅ **多重保护**：高程差 + 坡角 + 邻域完整性三重检验

#### 9.1.2 种子点选取策略

种子点是整个算法的"锚点"，其质量直接影响最终结果：

```
1. 投影到 XY 平面，划分 M×N 粗格网（grid_size 通常 = 3×cell_size）
2. 每个格网内取 Z 最低的点 → 候选种子集 S_candidate
3. 对 S_candidate 进行二次筛选：
   a. 计算全局高程直方图，确定主峰值（地面高程集中区间）
   b. 移除偏离主峰过远的候选点（可能是深坑或异常低点）
4. 最终种子集 S_seed ⊆ S_candidate
```

**关键参数影响**：
- `seed_grid_size` 过小 → 种子过多，可能混入非地面点
- `seed_grid_size` 过大 → 种子过少，可能导致漏分大片地面区域
- 推荐：自动模式（设为 0）通常效果良好

#### 9.1.3 迭代扩张机制

```
初始化: G = S_seed（已知地面点集），U = P \ G（未分类点集）

repeat max_iterations 次:
    new_grounds = ∅
    for each point p ∈ U:
        N_g = K_nearest_neighbors(p, G)   // 在 G 中查找 K 近邻
        
        if |N_g| < min_ground_neighbors:
            continue   // 邻域地面点不足，跳过
            
        // 计算扇区完整度（将邻域划分为 8 个 45° 扇区）
        sectors = partition_to_8_sectors(N_g)
        completeness = count_non_empty(sectors) / 8
        
        if completeness < min_neighbor_completeness:
            continue   // 边缘区域，不可靠
            
        // 计算参考高程（根据 reference_mode 选择方法）
        h_ref = compute_reference_elevation(N_g, mode)
        
        // 计算实际高程差和坡角
        Δh = p.z - h_ref
        slope = atan2(Δh, horizontal_distance(p, N_g))
        
        if Δh ≤ max_height_diff AND slope ≤ max_slope_deg:
            new_grounds.add(p)
    
    G = G ∪ new_grounds
    U = U \ new_grounds
    
    if new_grounds 为空:
        break   // 收敛，提前终止
```

### 9.2 IDW 插值数学原理

反距离加权（Inverse Distance Weighting）是经典的确定性插值方法：

$$H(x,y) = \frac{\sum_{i=1}^{n} w_i \cdot z_i}{\sum_{i=1}^{n} w_i}, \quad w_i = \frac{1}{d_i^p}$$

其中：
- $(x,y)$：待插值的栅格中心坐标
- $(x_i, y_i, z_i)$：第 $i$ 个已知地面点
- $d_i = \sqrt{(x-x_i)^2 + (y-y_i)^2}$：欧氏距离
- $p$：距离幂次（通常取 2）

**特点**：
- 距离越近的点权重越高
- 当 $p \to \infty$ 时退化为最近邻插值
- 当 $p = 1$ 时为线性距离反比
- 当 $p = 2$ 时为标准的反距离平方（默认）

### 9.3 空洞填充收敛性分析

空洞填充算法保证在有限步内收敛：

**证明概要**：
1. 每次迭代仅将 NoData 单元替换为有限邻域的均值
2. 替换后的值必然落在邻域最小值和最大值之间
3. 因此不会引入新的极值点
4. 由于栅格大小有限，最多经过 $R \times C$ 次迭代必收敛（$R,C$ 为行列数）

**实际收敛速度**：
- 典型 DEM 数据：5~20 次迭代
- 大面积连续空洞：可能需要 30~50 次
- 破碎空洞（孤岛状）：通常 3~8 次即收敛

## 12. 故障排除

### 12.1 常见错误及解决方案

#### 错误 1：无法打开 PLY 文件

```
Error: Cannot open input file: xxx.ply
```

**原因**：文件不存在、路径错误或权限不足

**解决方案**：
- 检查文件路径是否正确（注意相对路径是基于工作目录而非配置文件位置）
- 确认文件未被其他程序锁定
- 尝试使用绝对路径

#### 错误 2：PL Y 格式不支持

```
Error: Only ASCII PLY format is currently supported
```

**原因**：输入的是二进制 PLY 文件

**解决方案**：
- 使用 CloudCompare 或 PDAL 将二进制 PLY 转换为 ASCII 格式：
  ```bash
  pdal translate input_binary.ply output_ascii.ply --writers.ply.format="ascii"
  ```
- 或等待后续版本支持二进制 PLY（计划中）

#### 错误 3：缺少 x/y/z 属性

```
Error: PLY file missing required vertex property: z
```

**原因**：PLY 文件不包含完整的 XYZ 坐标

**解决方案**：
- 检查 PLY 文件的 header 部分，确认有 `property float x/y/z`
- 可能是法向量点云（仅有 normal_x/y/z），需转换为顶点坐标

#### 错误 4：内存不足

```
Error: std::bad_alloc / Out of memory
```

**原因**：点云过大，超出可用内存

**解决方案**：
- 启用 Tile 分块模式：`--set tile.mode=forced`
- 减小块大小：`--set tile.target_points_per_tile=1000000`
- 关闭不必要的应用释放内存

#### 错误 5：GeoTIFF 写入失败

```
Error: Failed to write GeoTIFF via Python backend
```

**原因**：Python 环境缺失或 rasterio 未正确安装

**解决方案**：
```powershell
# 检查 Python 和 rasterio
python --version
python -c "import rasterio; print(rasterio.__version__)"

# 重新安装依赖
pip install --upgrade rasterio numpy
```

#### 错误 6：地面点数量为 0

```
Warning: Ground filtering produced 0 ground points
```

**原因**：参数过于严格或点云特性不匹配

**解决方案**：
- 放宽 `max_height_diff`（如从 1.0 改为 3.0）
- 放宽 `max_slope_deg`（如从 20 改为 35）
- 增大 `max_iterations`（如从 6 改为 10）
- 检查点云是否已经是高程归一化的 DSM（数字表面模型）

#### 错误 7：DEM 全部为 NoData

```
Warning: DEM raster is entirely NoData
```

**原因**：地面点太少或分布过于稀疏

**解决方案**：
- 检查 `ground.ply` 确认是否有足够的地面点
- 减小 `dem.cell_size` 增加栅格密度
- 增大 `nearest_max_distance` 或 `idw_radius` 扩展搜索范围
- 若点云本身密度就很低，考虑先进行点云加密或融合

## 13. 已知限制

### 13.1 当前版本限制

| 限制 | 影响 | 计划 |
|------|------|------|
| 仅支持 ASCII PLY | 二进制 PLY 无法处理 | v0.2 计划支持 |
| 无原生 GeoTIFF 后端 | 依赖 Python 环境 | v0.2 计划接入 libtiff |
| 无 GPU 加速 | 大数据集处理较慢 | 远期规划 |
| 无并行文件 I/O | 磁盘可能成为瓶颈 | 远期规划 |
| 单线程 Tile 处理 | 多核利用率不足 | v0.3 计划优化 |

### 13.2 算法局限

| 局限 | 说明 | 建议 |
|------|------|------|
| 陡峭地形适应性 | > 35° 坡面可能欠分割 | 适当放宽 max_slope_deg |
| 悬垂结构处理 | 桥梁/悬挑可能被误判为地面 | 后处理人工修正 |
| 密集城区 | 紧邻建筑的地面点可能被遮挡 | 减小 seed_grid_size |
| 水体区域 | 水面高程波动大 | 后处理平滑或掩膜 |

## 附录 B：术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| DEM | Digital Elevation Model | 数字高程模型，表示地表高程的栅格数据 |
| DSM | Digital Surface Model | 数字表面模型，包含所有物体（建筑、植被）顶部高程 |
| PLY | Polygon File Format | 一种常用的三维点云/网格存储格式 |
| GeoTIFF | Georeferenced Tagged Image File Format | 含地理空间参考信息的 TIFF 图像格式 |
| EPSG | European Petroleum Survey Group | 坐标参考系编码标准 |
| CRS | Coordinate Reference System | 坐标参考系，定义地理位置的数学模型 |
| KNN | K-Nearest Neighbors | K 近邻算法，查找距离最近的 K 个点 |
| IDW | Inverse Distance Weighting | 反距离加权插值方法 |
| KD-Tree | K-Dimensional Tree | 一种高效的多维空间索引数据结构 |
| NoData | No Data | 表示无效或缺失数据的特殊标记值 |
| Tile | 瓦片/分块 | 将大数据集切分为小块进行处理的技术 |
| TIN | Triangulated Irregular Network | 不规则三角网，另一种 DEM 表达形式 |
| MAD | Median Absolute Deviation | 中位数绝对偏差，一种鲁棒统计量 |
| IQR | Interquartile Range | 四分位距，Q3-Q1 |
| WKT | Well-Known Text | 坐标参考系统的文本描述格式 |
