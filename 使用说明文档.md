## 1. 项目概述

### 1.1 功能定位

本工具是一个**命令行 DEM（数字高程模型）生成程序**，专门面向**光学测绘卫星摄影测量点云**数据处理。核心功能是从输入的 PLY 格式点云中提取地面点，并通过插值算法生成规则格网的高程栅格数据。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **输入格式** | ASCII PLY 点云文件（支持 XYZ + 可选强度/颜色/分类属性） |
| **输出格式** | GeoTIFF（地理参考）、PNG（可视化）、PLY（中间点云）、JSON 报告 |
| **预处理链路** | NaN/Inf 清洗 → 精确去重 → 极端离群点剔除（分位数统计） |
| **离群点过滤** | 基于 KNN 统计的 3D 邻域 SOR（Statistical Outlier Removal），支持鲁棒模式 |
| **地面滤波** | 迭代式种子扩张算法，支持三种参考模式：局部平面拟合 / IDW 加权 / 局部最低点 |
| **邻域完整性** | 8 扇区空间分布完整度校验，防止边缘误分类 |
| **DEM 插值** | 最近邻 / IDW（反距离加权）双模式，支持最大/最少参与点数控制 |
| **支撑栅格体系** | 直落格 → 支撑栅格精炼 → 有效域掩膜 → 对象掩膜的分层构建 |
| **空洞填充** | 受限半径邻域均值填充，支持最少邻居数约束 |
| **DTM 产物** | 分析型裸地 DTM + 展示型全域补全平滑 DTM + 高起伏对象掩膜 |
| **边缘处理** | 结构性边缘掩码（外边界/孔洞边界/对象边界/Tile 接缝风险区）+ 全局边缘收缩 |
| **分块处理** | Tile 模式自动切分大数据集（含缓冲区重叠，内存感知触发） |
| **配置管理** | JSON 配置 + 运行时 `--set` 参数覆盖（支持点分隔嵌套路径） |
| **批量处理** | Python 脚本支持目录级批量作业，可指定并发数 |

## 4. 构建指南

### 4.1 标准构建流程

```powershell
# 创建构建目录并配置
cmake -S . -B build -G "MinGW Makefiles"

# 编译（使用 4 个并行任务加速）
cmake --build build -j 4
```

构建完成后，可执行文件位于：
- `build/dem_cli.exe`（Windows）
- `build/dem_cli`（Linux）

### 4.2 高级构建选项

#### 启用/禁用 OpenMP 并行加速

```powershell
# 默认启用 OpenMP（编译器支持时自动开启）
cmake -S . -B build -DDEM_ENABLE_OPENMP=ON

# 禁用 OpenMP（调试或兼容性需求）
cmake -S . -B build -DDEM_ENABLE_OPENMP=OFF
```

#### 启用/禁用单元测试

```powershell
# 默认构建测试
cmake -S . -B build -DDEM_ENABLE_TESTS=ON

# 跳过测试构建以加快编译速度
cmake -S . -B build -DDEM_ENABLE_TESTS=OFF
```

#### 选择 GeoTIFF 后端

```powershell
# AUTO: 自动检测（默认优先 Python rasterio 后端）
cmake -S . -B build -DDEM_GTIFF_BACKEND=AUTO

# PYTHON: 强制使用 Python rasterio 后端
cmake -S . -B build -DDEM_GTIFF_BACKEND=PYTHON

# NATIVE: 使用原生 libtiff + PROJ（需预先安装相关开发包）
cmake -S . -B build -DDEM_GTIFF_BACKEND=NATIVE
```

## 5. 配置文件详解

配置文件采用 JSON 格式，完整示例见 `config/example_config.json`。以下为所有配置项的详细说明及代码中的实际默认值。

> ⚠️ **重要提示**：当参数值为 `0.0` 或空字符串时，系统会根据 `cell_size` 等**自动计算合理默认值**。具体归一化规则见[第 5.6 节末尾](#自动归一化规则)。

### 5.1 `input` — 输入文件配置

```json
{
  "input": {
    "file_path": "../data/基准.ply"
  }
}
```

| 参数 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|------|------|
| `file_path` | string | — | ✅ | 输入 ASCII PLY 文件路径（支持相对路径和绝对路径） |

**支持的 PLY 格式要求**：
- 仅支持 **ASCII** 格式（二进制 PLY 尚未实现）
- 必须包含 `x`、`y`、`z` 三个顶点属性
- 可选属性：`intensity`（强度）、`red`/`green`/`blue`（颜色）、`classification`（分类标签）、`normal_x`/`normal_y`/`normal_z`（法向量）

---

### 5.2 `crs` — 坐标参考系配置

```json
{
  "crs": {
    "epsg_code": 4326,
    "allow_unknown_crs": false
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `epsg_code` | int | 0 | EPSG 编码（`4326`=WGS84 经纬度，`3857`=Web Mercator，`4490`=CGCS2000 经纬度，`4547`=CGCS2000 3°带投影） |
| `crs_wkt` | string | "" | WKT 格式的 CRS 字符串（与 epsg_code 二选一） |
| `crs_wkt_file` | string | "" | WKT 文件路径（从外部文件加载 WKT 定义） |
| `allow_unknown_crs` | bool | false | 是否允许未知 CRS 继续处理 |

**常用 EPSG 代码**：

| EPSG | 名称 | 适用场景 |
|------|------|----------|
| 4326 | WGS84 经纬度 | 全球通用，单位：度 |
| 3857 | Web Mercator | Web 地图，单位：米 |
| 4490 | CGCS2000 经纬度 | 中国测绘通用，单位：度 |
| 4547 | CGCS2000 3°带 37 带 | 中国工程测量，单位：米 |

⚠️ 生产环境建议将 `allow_unknown_crs` 设为 `false` 以确保地理参考正确性。三者至少提供一个已知的 CRS 定义（epsg_code > 0 或非空 crs_wkt 或指定 crs_wkt_file），否则在 `allow_unknown_crs=false` 时会报错退出。

### 5.3 `preprocess` — 预处理参数

```json
{
  "preprocess": {
    "sample_spacing_count": 512,
    "max_extreme_sample": 5000
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `sample_spacing_count` | std::size_t | 1024 | 用于估算平均点间距的采样点数 |
| `max_extreme_sample` | std::size_t | 50000 | 极端离群点剔除时的最大采样数 |

**预处理阶段执行的操作**（按顺序）：

1. **NaN/Inf 清洗**：移除坐标包含 NaN 或 Inf 的点
2. **去重**：基于 XYZ 三维坐标精确匹配，仅保留首次出现的点
3. **极端离群点剔除**：对每个坐标轴计算分位数范围（Q1 - 3×IQR ~ Q3 + 3×IQR），超出范围的点被剔除
4. **重新索引**：为剩余点分配连续的全局索引号
5. **更新边界和密度**：重新计算空间范围和点密度
6. **估算点间距**：随机采样部分点，计算到最近邻的平均距离

### 5.4 `outlier` — KNN 离群点过滤参数

```json
{
  "outlier": {
    "knn": 16,
    "stddev_multiplier": 1.5,
    "use_robust_stat": false
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `knn` | std::size_t | 16 | 每个点的 K 近邻数量（≥ 1） |
| `stddev_multiplier` | double | 1.5 | 标准差倍数阈值系数 |
| `use_robust_stat` | bool | false | 是否使用鲁棒统计（中位数 + MAD 替代均值和标准差） |

**算法原理**（Statistical Outlier Removal, SOR）：

1. 为全部点构建 3D KD-Tree 空间索引
2. 查找每个点的 K 个最近邻
3. 计算该点到所有邻居的平均欧氏距离 d_i
4. 计算全局距离的均值 μ 和标准差 σ（或鲁棒模式下使用中位数和中位数绝对偏差 MAD）
5. 若 d_i > μ + multiplier × σ，则判定为离群点并移除

**参数调优建议**：

| 场景 | 推荐调整 |
|------|----------|
| 点云密度高 | 增大 `knn`（如 20~30） |
| 噪声较多 | 减小 `stddev_multiplier`（如 1.0~1.2） |
| 存在大量异常值 | 启用 `use_robust_stat = true` |

### 5.5 `ground` — 地面滤波核心参数 ⭐

这是**最关键**的配置段，直接影响地面点提取质量。

```json
{
  "ground": {
    "seed_grid_size": 0.0,
    "knn": 8,
    "search_radius": 0.0,
    "max_height_diff": 1.0,
    "max_slope_deg": 20.0,
    "max_iterations": 6,
    "min_ground_neighbors": 3,
    "reference_mode": "local_plane",
    "reference_fallback_mode": "inverse_distance_squared",
    "distance_weight_power": 2.0,
    "min_neighbor_completeness": 0.5
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `seed_grid_size` | double | 0.0 | 粗格网种子选取的格网尺寸（米）。`0`=自动按 cell_size×3 计算 |
| `knn` | std::size_t | 8 | 地面点迭代扩张时查找的近邻数量（≥ 1） |
| `search_radius` | double | 0.0 | 邻域搜索半径（米）。`0`=自动按 max(cell_size×6, seed_grid_size×2) 计算 |
| `max_height_diff` | double | 1.0 | 候选点到参考高程的最大允许高度差（米） |
| `max_slope_deg` | double | 20.0 | 最大允许坡角（度） |
| `max_iterations` | std::size_t | 6 | 迭代式地面点扩张的最大轮次（≥ 1） |
| `min_ground_neighbors` | std::size_t | 3 | 判定地面点所需的最小地面近邻数 |
| `reference_mode` | string | "local_plane" | 首选参考模式 |
| `reference_fallback_mode` | string | "inverse_distance_squared" | 首选模式失败时的回退参考模式 |
| `distance_weight_power` | double | 2.0 | IDW 权重的距离幂指数（> 0） |
| `min_neighbor_completeness` | double | 0.5 | 8 扇区邻域完整度阈值 [0, 1] |

**参考模式说明**：

| 模式值 | 枚举名 | 说明 |
|--------|--------|------|
| `"local_plane"` | LocalPlane | 最小二乘平面拟合（精度最高，计算量较大） |
| `"inverse_distance_squared"` / `"idw2"` | InverseDistanceSquared | IDW 反距离平方加权平均（平衡方案） |
| `"local_min"` | LocalMin | 取邻域内最低 Z 值（最保守，适合平坦地形） |

**地面滤波算法流程**：

**子阶段 A — 种子点选取（`seedGroundPoints`）**：
1. 将点云投影到 XY 平面
2. 按 `seed_grid_size` 划分粗格网
3. 在每个格网单元内选取 Z 值最低的点作为候选种子
4. 过滤掉明显偏高（可能是建筑物/树木顶部）的候选点
5. 得到初始地面种子点集

**子阶段 B — 迭代式地面点扩张（`extractGround`）**：

```
for iteration = 1 to max_iterations:
    for each 未分类点 p:
        1. 在已确认地面点中查找 K 个 2D 最近邻
        2. 若地面近邻数 < min_ground_neighbors → 跳过
        3. 计算 8 扇区邻域完整度
        4. 若完整度 < min_neighbor_completeness → 跳过（边缘区域不判定）
        5. 根据 reference_mode 计算 p 的参考高程 h_ref
        6. 计算高度差 Δh = p.z - h_ref
        7. 计算坡度 angle = atan(Δh / horizontal_distance)
        8. 若 Δh ≤ max_height_diff 且 angle ≤ max_slope_deg:
               标记 p 为地面点
           else:
               标记 p 为非地面点
    
    若本轮无新地面点加入 → 提前终止
```

**8 扇区邻域完整性**：以候选点为中心将 360° 方向划分为 8 个 45° 扇区，检查各扇区内是否有地面近邻点分布。完整度为有分布的扇区比例，用于防止数据边缘处的误分类。

**参数调优建议**：

| 场景 | 推荐调整 |
|------|----------|
| 平坦城市区域 | `max_height_diff`=0.3~0.5, `max_slope_deg`=10~15 |
| 山地丘陵 | `max_height_diff`=1.5~2.5, `max_slope_deg`=25~35 |
| 密集植被区 | `max_iterations`=8~10, `knn`=12~15 |
| 低密度点云 | `seed_grid_size` 手动设较大值（如 10~20 米） |
| 复杂建筑群 | `min_ground_neighbors`=5, `min_neighbor_completeness`=0.7 |

### 5.6 `dem` — DEM 插值参数

```json
{
  "dem": {
    "cell_size": 8.0,
    "nearest_max_distance": 0.0,
    "idw_radius": 0.0,
    "idw_max_points": 12,
    "idw_min_points": 3,
    "idw_power": 2.0,
    "nodata": -9999.0,
    "edge_shrink_cells": 1.0,
    "enable_edge_mask": true,
    "fill_holes": true,
    "fill_max_radius": 2,
    "fill_min_neighbors": 4
  }
}
```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `cell_size` | double | 1.0 | DEM 栅格分辨率（米/像素，> 0） |
| `nearest_max_distance` | double | 0.0 | 最近邻插值的最大搜索半径（米）。`0`=自动按 max(cell_size×6, search_radius) 计算 |
| `idw_radius` | double | 0.0 | IDW 插值的搜索半径（米）。`0`=自动按 max(cell_size×8, search_radius + 2×cell_size) 计算 |
| `idw_max_points` | std::size_t | 12 | IDW 插值最多使用的邻近点数 |
| `idw_min_points` | std::size_t | 3 | IDW 插值最少需要的邻近点数（不足则标记 NoData） |
| `idw_power` | double | 2.0 | IDW 距离权重幂指数（> 0） |
| `nodata` | double | -9999.0 | 无数据标记值 |
| `edge_shrink_cells` | double | 1.0 | 全局边缘收缩的单元格数 |
| `enable_edge_mask` | bool | true | 是否启用结构性边缘掩码 |
| `fill_holes` | bool | true | 是否启用空洞填充 |
| `fill_max_radius` | int | 2 | 空洞填充最大搜索半径（单元格，≥ 0） |
| `fill_min_neighbors` | std::size_t | 4 | 空洞填充最少需要的有效邻居数 |

**DEM 栅格产物体系**（按生成顺序）：

| 产物名称 | 生成方法 | 说明 |
|----------|----------|------|
| `dem_raw_direct` | rasterizeMinimum | 预处理后**所有有效点**的直接落格（每格取最低 Z） |
| `dem_ground_direct` | rasterizeMinimum | **地面点**的直接落格（每格取最低 Z） |
| `dem_support_mask` | buildSupportMask | 二值掩膜：1=地面点直接支撑的像元 |
| `dem_mask`（最终有效域） | buildDomainMask | 基于 accepted support 和对象掩膜构建的最终 DEM 有效域 |
| `dem_nearest` | interpolateNearest | 在有效域内使用**最近邻**方法插值的 DEM |
| `dem_idw` | interpolateIdw | 在有效域内使用 **IDW** 方法插值的 DEM |
| `dem_nearest_filled` | fillHoles(dem_nearest) | 最近邻 DEM 经过空洞填充后的结果 |
| `dem_idw_filled` | fillHoles(dem_idw) | IDW DEM 经过空洞填充后的结果 |
| `dtm_object_mask` | buildObjectMask | 基于 raw-ground 残差检测的建筑/树木等高起伏对象掩膜 |
| `dtm_analysis` | buildAnalysisDtm | 分析型裸地 DTM（仅在裸地域上计算） |
| `dtm_display` | buildDisplayDtm | 展示型全域补全 + 平滑 DTM（内部空洞会被全局重建后再平滑） |
| `dem_edge_mask` | buildEdgeMask | 结构性边缘掩码（外边界/孔洞边界/填洞前沿/Tile 接缝） |

**支撑栅格精炼（refineAcceptedSupport）**：从原始直落格和地面直落格的差异中推断出可接受的支撑区域，补充那些没有地面点直接覆盖但根据上下文判断应为地面的像元。

**分辨率选择建议**：

| 用途 | 推荐 cell_size |
|------|----------------|
| 高精度制图 | 1.0 ~ 2.0 m |
| 中等精度 | 5.0 ~ 10.0 m |
| 概览/背景 | 20.0 ~ 50.0 m |

#### 自动归一化规则

当以下参数值为 `0.0`（表示"自动"）时，系统按照以下规则归一化（顺序依赖，先计算的值被后续使用）：

| 参数 | 归一化公式 |
|------|-----------|
| `ground.seed_grid_size` | `cell_size × 3` |
| `ground.search_radius` | `max(cell_size × 6, seed_grid_size × 2)` |
| `dem.nearest_max_distance` | `max(cell_size × 6, search_radius)` |
| `dem.idw_radius` | `max(cell_size × 8, search_radius + 2 × cell_size)` |
| `tile.tile_buffer` | `max(search_radius, nearest_max_distance, idw_radius) + fill_max_radius × cell_size` |

```json

  }

```

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `mode` | string | "auto" | 分块模式：`"auto"` / `"disabled"` / `"false"` / `"forced"` / `"true"` |
| `tile_size_cells` | std::size_t | 1000 | 每个 Tile 的目标单元格数（约 1000×1000，≥ 1） |
| `tile_buffer` | double | 0.0 | Tile 缓冲区宽度（米）。`0`=自动计算（见上方归一化规则） |
| `memory_limit_mb` | std::size_t | 1024 | 触发自动分块的内存阈值（MB） |

**分块模式说明**：

| 模式值 | 行为 |
|--------|------|
| `"auto"` | 根据预估内存占用自动决定是否分块（超过 memory_limit_mb 时触发） |
| `"disabled"` / `"false"` | 始终不分块，强制单次全量处理 |
| `"forced"` / `"true"` | 始终启用分块，无论数据量大小 |

**Tile 结构**：每个 Tile 由主区域（main_bounds）和缓冲区域（buffered_bounds）组成。缓冲区确保 Tile 边界处的插值质量不受截断影响。各 Tile 分别处理后，通过最小值聚合策略合并为全局结果。

**何时需要分块**：
- 点云总点数超过数百万级
- 可用内存 < 8GB
- 处理超大面积（如整景卫星影像）

## 7. 处理流程详解

### 7.1 完整流水线架构

整个处理流程由 [MainController](include/dem/MainController.hpp) 编排，分为两大阶段：

**Support 阶段（`processSupportStage`）**：点级处理，只生成分类后的点云和直落格产物

**Finalize 阶段（`finalizeArtifacts`）**：栅格级处理，基于全局 support/domain 构建最终 DEM/DTM/QC 产物

```
┌───────────────────────────────────────────────────────────────┐
│                    MainController::run()                       │
│                                                                 │
│  ① ConfigManager.load()     加载+解析+校验+归一化配置            │
│  ② CRSManager.resolve()     解析坐标参考系                      │
│  ③ InputManager.validate()  输入验证（头部+扫描+权限）           │
│  ④ TileManager.shouldUseTiles()  决定是否分块                  │
│                                                                 │
│  ┌─ processSingleCloud() ─────┐  或  ┌─ processTiles() ──────┐ │
│  │                             │      │                        │ │
│  │  processSupportStage():     │      │  ① createTiles()       │ │
│  │  ├─ PreprocessEngine.run() │      │  ② materializeTiles()  │ │
│  │  ├─ GroundFilterEngine     │      │  ③ 逐 Tile:             │ │
│  │  │   .removeOutliers()     │      │     processSupportStage │ │
│  │  │   .extractGround()      │      │     processSingleCloud  │ │
│  │  ├─ DEMEngine.rasterize*() │      │  ④ mergeMinimumRaster   │ │
│  │  └─ DEMEngine.buildSupport │      └────────────────────────┘ │
│  │     Mask()                 │                               │
│  │                             │                               │
│  │  finalizeArtifacts():       │                               │
│  │  ├─ refineAcceptedSupport   │                               │
│  │  ├─ buildDomainMask        │                               │
│  │  ├─ buildObjectMask        │                               │
│  │  ├─ interpolateNearest     │                               │
│  │  ├─ interpolateIdw         │                               │
│  │  ├─ fillHoles (×2)         │                               │
│  │  ├─ buildAnalysisDtm       │                               │

└───────────────────────────────────────────────────────────────┘
```

### 7.2 各阶段详情

#### 阶段 1：输入验证（InputManager）

- **模块**: [InputManager](include/dem/InputManager.hpp)
- **功能**: 全面检查输入文件的合法性和数据质量
- **检查项**:
  - 文件是否存在且可读
  - 是否为 ASCII PLY 格式
  - 是否包含 x, y, z 属性
  - 点数是否合理（>0）
  - 采样扫描获取统计摘要（NaN/Inf 点数、重复点线索、边界范围、预估内存）
  - 输出目录写权限探测
- **产出**: `InputValidationReport`（含 `PlyHeader`、`PointCloudScanSummary`、警告列表）

#### 阶段 2：CRS 解析（CRSManager）

- **模块**: [CRSManager](include/dem/CRSManager.hpp)
- **功能**: 将用户提供的 CRS 配置统一转换为可用的 WKT 定义
- **支持格式**: EPSG 编码、WKT 字符串、WKT 外部文件
- **产出**: `CRSDefinition.resolved_wkt`（传递给 GeoTIFF 写入脚本）

#### 阶段 3：预处理（PreprocessEngine）

- **模块**: [PreprocessEngine](include/dem/PreprocessEngine.hpp)
- **步骤 3.1 — NaN/Inf 清洗** (`removeInvalidPoints`)
  - 逐点扫描 XYZ 坐标，移除任何包含 NaN 或 Inf 的点
  
- **步骤 3.2 — 去重** (`removeDuplicatePoints`)
  - 基于 XYZ 三维坐标精确匹配，仅保留首次出现的点
  
- **步骤 3.3 — 极端离群点剔除** (`removeExtremePoints`)
  - 在每个坐标轴上计算 Q1/Q3 分位数和 IQR
  - 移除超出 `[Q1 - 3×IQR, Q3 + 3×IQR]` 范围的点
  - 采样上限由 `preprocess.max_extreme_sample` 控制
  
- **步骤 3.4 — 重新索引 + 更新边界密度 + 估算点间距**
  - 分配连续索引号
  - 重算 Bounds 和点密度
  - 随机采样 `sample_spacing_count` 个点，计算到最近邻的平均距离

#### 阶段 4：KNN 离群点过滤（GroundFilterEngine::removeOutliers）

- **模块**: [GroundFilterEngine](include/dem/GroundFilterEngine.hpp)
- **算法**: Statistical Outlier Removal (SOR)
- **流程**:
  1. 构建 3D KD-Tree 索引（[SpatialIndexManager](include/dem/SpatialIndexManager.hpp)）
  2. 对每个点执行 KNN 查询
  3. 计算点到 K 个邻居的平均欧氏距离 d_i
  4. 计算全局 {d_i} 的均值 μ 和标准差 σ（或鲁棒模式下用中位数和 MAD）
  5. 若 d_i > μ + `stddev_multiplier` × σ → 标记为离群点
  6. 一次性移除所有离群点

#### 阶段 5：地面滤波（GroundFilterEngine::extractGround）

**子阶段 5.1 — 种子点选取（`seedGroundPoints`）**

- 按 `seed_grid_size` 在 XY 平面上划分规则格网
- 每个格网单元中选择 Z 值最小的点作为候选种子
- 过滤掉明显偏高的候选点（可能来自建筑物顶部或树冠）
- 最终得到初始地面种子点集

**子阶段 5.2 — 迭代式地面点扩张**

```
for iteration = 1 to max_iterations:
    构建当前地面点的 2D KD-Tree 索引
    
    for each 未分类点 p:
        a) 在地面点 2D 索引中 knn2D 查找 K 个最近地面邻居
        b) 若邻居数 < min_ground_neighbors → 跳过
        c) 计算 8 扇区邻域完整度
        d) 若完整度 < min_neighbor_completeness → 跳过
        e) 根据 reference_mode 计算 p 的参考高程 h_ref:
           - local_plane:     对邻居点做最小二乘平面拟合 z=ax+by+c
           - idw2:            加权平均 Σ(w_i * z_i) / Σ(w_i), w_i = 1/d^power
           - local_min:       取邻居中最小 z 值
           （若首选模式失败，自动回退到 fallback 模式）
        f) 计算 Δh = p.z - h_ref
        g) 计算水平距离 d_horizontal 和坡度 slope = atan(Δh / d_horizontal)
        h) 若 Δh ≤ max_height_diff 且 slope ≤ max_slope_deg:
               → 标记 p 为地面点，记录 reference_z, local_height_diff, local_slope
           else:
               → 标记 p 为非地面点
    
    记录本轮新增地面点数
    若本轮无新地面点 → break 提前终止
```

#### 阶段 6：支撑栅格构建（DEMEngine）

- **模块**: [DEMEngine](include/dem/DEMEngine.hpp)

**6.1 直落格（rasterizeMinimum）**：
- 对每个点计算所属栅格单元 (row, col)
- 每个单元记录落入其中的点的**最低 Z 值**
- 同时生成 valid_mask 标记有直接落点的单元

**6.2 支撑掩膜（buildSupportMask）**：
- 从 ground_direct 栅格生成二值掩膜
- `1` = 有地面点直接支撑的像元

**6.3 对象掩膜（buildObjectMask）**：
- 比较 raw_direct 和 ground_direct 的差异
- 残差显著偏高的像元标记为建筑/树木等高起伏对象候选

**6.4 支撑精炼（refineAcceptedSupport）**：
- 从 raw-ground 差异中推断可接受支撑区域
- 补充无地面点直接覆盖但上下文表明为地面的像元

**6.5 有效域掩膜（buildDomainMask）**：
- 基于 accepted support 和 object_mask 构建最终的 DEM 有效域
- 对象区域被排除在有效域之外

#### 阶段 7：DEM 插值（DEMEngine）

**7.1 最近邻插值（interpolateNearest）**：
- 对有效域内的每个 NoData 单元，在支撑栅格中搜索 `nearest_max_distance` 半径内的最近有效邻居
- 直接采用邻居的高程值（保持原始值不变，速度快）

**7.2 IDW 插值（interpolateIdw）**：
- 对有效域内的每个 NoData 单元，在 `idw_radius` 半径内搜索有效支撑点
- 使用 `idw_max_points` 个最近点，不少于 `idw_min_points` 个
- 加权平均：`z = Σ(z_i / d^power) / Σ(1 / d^power)`
- 结果更平滑，但可能过度平滑陡坎特征

#### 阶段 8：空洞填充（DEMEngine::fillHoles）

- **算法**: 受限半径邻域均值填充
- 对有效域内仍为 NoData 的单元，在 `fill_max_radius` 范围内搜索有效邻居
- 要求至少 `fill_min_neighbors` 个有效邻居
- 取邻居均值作为填充值
- 多次迭代直至收敛或达到隐含上限

#### 阶段 9：DTM 产物生成

**9.1 分析型 DTM（buildAnalysisDtm）**：
- 仅在裸地域（domain_mask \ object_mask）上保留精确值
- 对象区域和外部区域设为 NoData
- 用于定量分析和精度评估

**9.2 展示型 DTM（buildDisplayDtm）**：
- 在分析型 DTM 基础上进行全域补全和平滑
- 内部空洞（对象区域）会被全局重建填充
- 适用于可视化展示和制图输出

#### 阶段 10：边缘掩码（DEMEngine::buildStructuralEdgeMask）

识别以下结构性边缘区域：
- **外边界**：有效域的最外圈像素
- **大孔洞边界**：内部 NoData 区域的轮廓
- **对象边界**：建筑/树木对象的轮廓
- **Tile 接缝**（分块模式）：Tile 主区域交界处的高差异像素（接缝评分 > 全局均值 × 1.5）
- **填洞前沿**：填洞前后发生变化的区域

此外，全局边缘收缩（`applyGlobalEdgeShrink`）会将外缘 `edge_shrink_cells` 宽度的区域统一置为 NoData，消除边界效应。

#### 阶段 11：结果输出（OutputManager）

- **模块**: [OutputManager](include/dem/OutputManager.hpp)
- **点云输出**: 4 个 ASCII PLY 文件（filtered/seed/ground/nonground），含 classification 属性
- **栅格输出**: 通过 [write_geotiff.py](scripts/write_geotiff.py) 调用 Python `rasterio` 写入 GeoTIFF，同时生成 PNG
- **桥接机制**: C++ 将栅格数据序列化为 `.bin` 二进制文件 + `.meta.json` 元数据，再调用 Python 脚本完成实际写入
- **统计输出**: `stats.txt`（键值对格式）+ `report.json`（JSON 格式）
- **配置回写**: `config_used.txt`（记录实际使用的完整参数，便于复现）

## 9. 算法原理

### 9.1 空间索引（SpatialIndexManager）

基于 [nanoflann](https://github.com/jlblancoc/nanoflann) 库封装 KD-Tree，提供两种索引维度：

| 索引类型 | 维度 | 用途 |
|----------|------|------|
| 2D KD-Tree | X-Y | 地面滤波中的近邻搜索、DEM 插值 |
| 3D KD-Tree | X-Y-Z | 离群点检测中的邻域距离计算 |

查询接口：
- `knn2D(x, y, k)` / `knn3D(x, y, z, k)` — K 近邻查询
- `radius2D(x, y, r, max)` / `radius3D(x, y, z, r, max)` — 半径查询

### 9.2 地面滤波参考模式

**局部平面拟合（LocalPlane）**：

对 K 个地面近邻点 {(x_i, y_i, z_i)} 做最小二乘平面拟合：

```
z = a·x + b·y + c
```

通过正规方程求解系数 [a, b, c]^T，然后代入候选点坐标得到参考高程。
当近邻点近似共面或条件数过大时自动失败并回退到 IDW 模式。

**IDW 加权（InverseDistanceSquared）**：

```
h_ref = Σ(w_i · z_i) / Σ(w_i), 其中 w_i = 1 / d_i^power
```

d_i 为候选点到第 i 个地面邻居的水平距离，power 默认为 2（反距离平方）。

**局部最低点（LocalMin）**：

```
h_ref = min(z_1, z_2, ..., z_K)
```

最保守的策略，适合地形非常平坦的场景。

### 9.3 Tile 接缝检测

在分块模式下，`buildTileSeamMask` 通过以下步骤检测接缝风险：

1. 计算全局分析型 DTM 上所有相邻像素差的均值 `global_mean_diff`
2. 设定接缝阈值 `seam_threshold = global_mean_diff × 1.5`
3. 沿 Tile 主区域边界线逐像素比较跨边界高程差
4. 差异超过阈值的像素标记为接缝风险区
5. 输出接缝评分 `seam_score = seam_mean_diff / global_mean_diff`（越高表示接缝越明显）

### 9.4 空洞填充算法

采用迭代式受限邻域均值填充：

```
while 存在待填充的 NoData 单元:
    for each NoData 单元 cell:
        在 fill_max_radius 范围内搜索有效邻居
        if 邻居数 >= fill_min_neighbors:
            cell.value = mean(邻居值)
        else:
            保持 NoData（无法填充）
    
    if 本轮无变化 → 收敛，停止
```

## 10. 批量处理

### 10.1 使用 batch_process.py

```powershell
python .\scripts\batch_process.py \
    --cli .\build\dem_cli.exe \
    --config .\config\example_config.json \
    --input-dir .\data \
    --output-root .\output \
    --glob "*.ply" \
    --jobs 4
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--cli` | `build/dem_cli.exe` | dem_cli 可执行文件路径 |
| `--config` | `config/example_config.json` | 基础 JSON 配置模板 |
| `--input-dir` | `data` | 包含 PLY 文件的输入目录 |
| `--output-root` | `output` | 批量输出根目录 |
| `--glob` | `*.ply` | 文件名匹配模式 |
| `--jobs` | `1` | 并发处理数（1=串行） |

### 10.2 批量输出结构

```
output/batch_<YYYYMMDD_HHMMSS>/
├── configs/
│   ├── dataset_01.json    # 每个数据集的独立配置（自动修改了 input.file_path）
│   ├── dataset_02.json
│   └── ...
├── <dataset_name_1>/      # 每个数据集的独立输出（等同于单次运行的 output/<timestamp>/）
│   ├── dem/
│   ├── pointcloud/
│   └── log/
├── <dataset_name_2>/
│   └── ...
├── summary.json           # JSON 格式的汇总报告
└── summary.csv            # CSV 格式的汇总表（含状态/耗时/输出路径等）
```

### 10.3 注意事项

- 批量脚本会自动为每个数据集关闭 `timestamp_subdir`（避免多层时间戳嵌套）
- 数据集按文件大小降序排列处理（大的先做）
- 并发模式下各数据集独立运行，互不干扰
- 失败的数据集不会阻塞其他数据集的处理
- 最终汇总报告中会列出所有失败项

## 11. 性能优化建议

### 11.1 内存优化

| 策略 | 效果 |
|------|------|
| 启用 Tile 模式 | 大幅降低峰值内存占用（分块加载处理） |
| 减小 `tile.tile_size_cells` | 每个 Tile 的栅格更小，内存更低 |
| 降低 `preprocess.sample_spacing_count` | 减少点间距估算阶段的采样开销 |
| 关闭不必要的输出 | 减少 I/O 开销 |

### 11.2 计算速度优化

| 策略 | 效果 |
|------|------|
| 启用 OpenMP（`-DDEM_ENABLE_OPENMP=ON`） | KD-Tree 查询等并行加速 |
| 增大 `dem.cell_size` | 栅格总单元数减少，插值更快 |
| 减小 `ground.max_iterations` | 更早收敛终止 |
| 减小 `ground.knn` 和 `outlier.knn` | KNN 查询更快 |
| 使用 `ground.reference_mode = "local_min"` | 参考高程计算最快（无需平面拟合） |

### 11.3 I/O 优化

| 策略 | 效果 |
|------|------|
| 设置 `output.write_png = false` | 跳过 PNG 生成（减少约一半 I/O 时间） |
| 将输出目录放在 SSD 上 | 显著加快文件写入速度 |
| 关闭实时杀毒软件扫描输出目录 | 避免 I/O 阻塞 |

## 12. 故障排除

### 12.1 常见错误

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `Unable to open config file:` | 配置文件路径不存在或不可读 | 检查路径拼写，使用绝对路径 |
| `Config must define input.file_path.` | 配置缺少必填字段 | 在 JSON 中添加 `input.file_path` |
| `dem.cell_size must be > 0` | 分辨率未设置或非法 | 设置正数，如 `"cell_size": 8.0` |
| `CRS is required unless...` | 未提供 CRS 且不允许未知 | 设置 `crs.epsg_code` 或 `crs.allow_unknown_crs=true` |
| `Unable to open tile file:` | Tile 模式下暂存文件丢失 | 检查磁盘空间和临时目录权限 |
| `GeoTIFF writer script failed` | Python 环境问题 | 确认 `rasterio` 已安装，Python 路径正确 |
| `DEM cell size is finer than estimated point spacing` | 分辨率高于点密度 | 增大 `cell_size` 或接受稀疏支撑和空洞 |

### 12.2 日志排查

运行日志位于 `output/<timestamp>/log/run_log.txt`，包含：
- 每个 Info 级别的消息记录处理进度
- Warning 级别的消息提示潜在问题（如分辨率过细）
- Error 级别的消息记录致命错误

统计文件 `stats.txt` 中的关键指标：
- `cell_size_vs_spacing_ratio`：若 < 0.75 则分辨率相对于点密度过高
- `ground_direct_active_ratio`：地面点覆盖率，过低说明地面点稀疏
- `dem_mask_active_ratio`：最终有效域占比
- `tile_mode_used`：是否触发了分块模式

## 13. 已知限制

1. **仅支持 ASCII PLY**：二进制 PLY 格式读取尚未实现
2. **GeoTIFF 依赖 Python**：原生 libtiff + libgeotiff + PROJ 后端已设计但尚未接入，当前必须通过 Python 环境中的 `rasterio` 完成 GeoTIFF 写入
3. **Tile 模式暂存**：分块处理时通过 `output/tile/*.xyz` 文本文件暂存各 Tile 的点云数据，避免在最终汇总阶段一次性将全部结果点驻留内存
4. **单线程处理**：除 OpenMP 加速的 KD-Tree 查询外，主要处理逻辑为单线程；Tile 模式下的各 Tile 串行处理
5. **无增量处理**：每次运行都从头开始完整处理，不支持增量更新
6. **无 GUI 界面**：纯命令行工具，无图形用户界面
